Sistema de Gestion de Calidad de Software

Nuestro Proyecto:

El proyecto que estamos realizando es sobre Sistema de Gestión de Calidad de Software (SGCS) es el conjunto de políticas, procesos y prácticas que una organización aplica para asegurar que el software desarrollado cumpla con los requisitos de calidad establecidos, abarcando todo el ciclo de vida del producto desde la planificación hasta el mantenimiento; su finalidad es estandarizar el trabajo, prevenir y detectar errores, medir resultados con métricas, cumplir con normas internacionales y fomentar la mejora continua, lo que permite entregar software confiable, eficiente y alineado con las necesidades del cliente.

Resumen de Opciones:

Gestión de la Calidad del Software:

Asegura que el software sea confiable, eficiente, usable y mantenible, usando métricas, estándares (ISO 9126/25000) y pruebas (unitarias, integración, sistema, aceptación) con herramientas como JIRA, TestLink, Selenium y SonarQube.

Procesos de Desarrollo de Software:

Describe las etapas del SDLC (planificación, análisis, diseño, implementación, pruebas, despliegue y mantenimiento). Se pueden usar modelos tradicionales (Cascada, V, Iterativo/Incremental) o metodologías ágiles (Scrum, Kanban, XP) para organizar, controlar y mejorar el desarrollo del software.
