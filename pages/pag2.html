Técnicas de Aseguramiento de la Calidad

  1. Tipos de pruebas de software

      Los tipos de prueba se refieren al propósito principal de la prueba:

    • Pruebas funcionales: Verifican que el software cumple con los requisitos especificados.
      • Ejemplos: pruebas de funcionalidades, de integración de módulos, de aceptación del usuario.
    • Pruebas no funcionales: Evaluan aspectos no relacionados directamente con la funcionalidad.
      • Ejemplos: rendimiento, seguridad, usabilidad, compatibilidad, carga, estrés.
    • Pruebas de regresión: Verifican que cambios o correcciones no hayan introducido errores nuevos.
    • Pruebas de aceptación: Validan que el software cumple con las expectativas del cliente o usuario final.
  2. Niveles de pruebas de software

      Se refiere a cuándo y dónde se realizan las pruebas dentro del ciclo de desarrollo:

    1. Pruebas unitarias (Unit Testing):
      • Nivel más bajo.
      • Verifica que cada módulo o componente funcione correctamente de manera aislada.
    2. Pruebas de integración (Integration Testing):
      • Evalúa la interacción entre módulos o componentes combinados.
      • Busca errores en interfaces y comunicación entre módulos.
    3. Pruebas del sistema (System Testing):
      • Verifica el sistema completo como un todo.
      • Abarca tanto aspectos funcionales como no funcionales.
    4. Pruebas de aceptación (Acceptance Testing):
      • Último nivel antes de entregar el software al usuario.
      • Confirma que el producto cumple con los requisitos y expectativas del cliente.
  1. Técnicas de pruebas de software

      Las técnicas indican cómo se diseñan y ejecutan las pruebas:

    1. Técnicas de caja negra (Black Box)
      • Se enfocan en la entrada y salida del software, sin considerar la estructura interna.
      • Ejemplos:
        • Pruebas de equivalencia (Equivalence Partitioning)
        • Análisis de valores límite (Boundary Value Analysis)
        • Pruebas de casos de uso (Use Case Testing)
    2. Técnicas de caja blanca (White Box)
      • Se basan en el código y la lógica interna del software.
      • Ejemplos:
        • Cobertura de sentencias (Statement Coverage)
        • Cobertura de ramas (Branch Coverage)
        • Pruebas de caminos (Path Testing)
    3. Técnicas basadas en experiencia o exploratorias
      • Se basan en la intuición y experiencia del tester, sin seguir un guion estricto.
      • Ejemplos:
        • Pruebas exploratorias
        • Pruebas ad hoc
