Herramientas de Calidad

  1. Modelo en Cascada (Waterfall)

    2. Es un modelo secuencial donde cada fase debe completarse antes de pasar a la siguiente. Fases típicas: requisitos → diseño → implementación → pruebas → mantenimiento.

    Ventajas

    • Estructura clara y fácil de entender.
    • Documentación abundante en cada fase.
    • Bueno para proyectos pequeños y bien definidos.
    • Permite control estricto del avance.

    Desventajas

    • Muy rígido: no admite cambios una vez definida una fase.
    • Si hay errores en requisitos, se descubren tarde.
    • Poco feedback del cliente durante el proceso.
    • Riesgo alto si el proyecto es largo o complejo.

    Cuándo usarlo

    • Proyectos pequeños o con requisitos muy claros y estables.
    • Cuando el cliente sabe exactamente lo que quiere.
    • Entornos donde se requiere mucha documentación formal.
  2. Modelo en V

    3. Es una evolución del cascada, pero con mayor énfasis en la verificación y validación. Cada fase de desarrollo tiene su fase de prueba correspondiente (ej.: requisitos ↔ pruebas de aceptación, diseño ↔ pruebas de integración, etc.).

    Ventajas

    • Se asegura la calidad desde el inicio.
    • Enfatiza las pruebas y validación en cada etapa.
    • Útil para proyectos críticos (médicos, aeroespaciales, militares).

    Desventajas

    • Igual de rígido que cascada: difícil gestionar cambios.
    • Mucho esfuerzo en planificación y documentación.
    • No se adapta bien a requisitos cambiantes.

    Cuándo usarlo

    • Sistemas críticos que requieren alta fiabilidad.
    • Proyectos donde los errores deben detectarse lo antes posible.
    • Cuando los requisitos son claros y poco cambiantes.
  3. Modelo Iterativo

    4. El sistema se desarrolla en versiones parciales que se van refinando con cada iteración. Se empieza con una funcionalidad básica y se mejora gradualmente.

    Ventajas

    • Permite ver resultados tempranos.
    • El cliente puede dar feedback en cada iteración.
    • Flexible ante cambios en requisitos.
    • Reduce riesgos al validar en etapas tempranas.

    Desventajas

    • Puede ser más costoso que cascada.
    • Requiere una gestión y comunicación constante.
    • Si no se planifica bien, puede generar iteraciones interminables.

    Cuando usarlo

    • Proyectos con requisitos poco claros o cambiantes.
    • Cuando se necesita mostrar prototipos tempranos al cliente.
    • Desarrollos donde la retroalimentación continua es clave.
  1. Modelo Incremental

    2. El sistema se construye en módulos o incrementos. Cada incremento añade nuevas funcionalidades hasta completar el producto final. Similar al iterativo, pero con entregas funcionales completas por módulo.

    Ventajas

    • Se entregan partes utilizables rápidamente.
    • Flexibilidad para agregar o cambiar funcionalidades.
    • Menor riesgo, porque el cliente prueba versiones funcionales desde el inicio.
    • Los errores se detectan pronto en cada módulo.

    Desventajas

    • Puede complicarse la integración de módulos.
    • Se requiere buena planificación modular.
    • Si los incrementos no están bien definidos, el sistema puede quedar inconsistente.

    Cuando usarlo

    • Proyectos grandes que pueden dividirse en partes claras.
    • Cuando se necesita entregar valor rápido al cliente.
    • Desarrollo de software empresarial, web o apps con múltiples módulos.
